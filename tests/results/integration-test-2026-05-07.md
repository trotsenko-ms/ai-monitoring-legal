## 4. ВИЯВЛЕНІ ОСОБЛИВОСТІ ТА НОТАТКИ

1. **Затримка індексації на kmu.gov.ua** — документи з датою публікації D можуть з'явитися в пошуку через 1-2 дні. Monitor-kmu може захопити документи за межами period.from. Validator коректно позначає їх як `uncertain`. Рекомендація: при формуванні period.from для kmu — відраховувати від 48 годин тому замість точного часу попереднього запуску.

2. **itd.rada.gov.ua** — сайт ВРУ може бути тимчасово недоступним (технічні роботи). Monitor-rada-bills коректно обробляє source_unavailable і повертає items[]. Orchestrator продовжує роботу з двома іншими джерелами.

3. **Перекриття контенту між гілками** — один і той самий документ (наприклад, Постанова КМУ № 534) може потрапити як до monitor-kmu (гілка держорганів), так і до monitor-edo (через zakon.rada.gov.ua). Це очікувана поведінка — дедуплікація per-agent файлами не охоплює між-гілкову дедуплікацію. Дублювання в email-дайджестах прийнятне для пілоту.

4. **Формат email-дайджесту** — перевірено структуру: секції по джерелах (ВРУ / КМУ / НБУ) формуються коректно; секції з 0 документів містять «Нових документів не виявлено.»; uncertain items виведено в окремій секції з validation_note.
